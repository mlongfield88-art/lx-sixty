# Security Policy

LX Sixty Group takes security seriously. This document explains how to report a vulnerability you find in any of our public properties.

## Reporting a vulnerability

Email **myles@lxsixty.com** with a clear description of the issue, the affected URL or component, and steps to reproduce.

Please **do not** open a public GitHub issue for security findings. Email gets a faster, more confidential response.

## Scope

This policy covers the following properties:

- `lxsixty.com` (LX Sixty Group corporate site)
- `proluxetravels.com` (ProLuxe Travel agency site)
- `sangnoir.co.uk` (Sangnoir Risk Management site)
- `casa-marenya.proluxetravels.com` (Casa Marenya single-property site)
- Public-facing pages of `capsian-invest.lxsixty.com`

Out of scope:

- Internal tools (proposals generator, IBA portal staff dashboard)
- Third-party services we depend on (Cloudflare Pages, Firebase, GitHub, Formspree, Google Workspace)
- Brute-force attacks against authentication systems
- Findings from automated scanners that haven't been verified to be exploitable

## What to expect

- Acknowledgement within 3 working days
- Assessment and remediation timeline within 10 working days for confirmed vulnerabilities
- Credit in this file (or a separate hall of fame) for valid reports if you wish

## Safe harbour

Good-faith security research conducted within the scope above will not result in legal action. Avoid:

- Privacy violations (do not access, modify, or destroy data that is not yours)
- Service disruption (no DoS testing)
- Social engineering of staff
- Physical security testing

## Out-of-band contact

For RFC 9116 compliant tooling: `https://lxsixty.com/.well-known/security.txt`
